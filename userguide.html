Quick Start

  1. Upload MP3, WAV, or FLAC.
  2. Select mode: Draft for speed, HQ for stronger processing quality.
  3. Track per-stage progress in dashboard status rows.
  4. Open editor views and apply note, timing, and revision controls.
  5. Export and review generated artifacts.

Standard Entrypoints (Windows/Linux)

Use repository-root wrappers for consistent startup behavior:

./start.sh --mode draft --host 127.0.0.1 --port 4173
.\start.ps1 -mode hq -host 127.0.0.1 -port 4173

Both wrappers delegate to infrastructure/local-dev/start_transcriberator.py, which starts a local dashboard server for interactive audio upload and transcription runs.

  • --smoke-run --json: emit machine-readable smoke-run output.
  • --smoke-run --fail-stage <stage-name>: simulate startup failures for troubleshooting.
  • --no-hq-degradation: disable HQ fallback so separation failures fail fast.

System Modes

Draft: fastest deterministic pass for iteration loops.

HQ: separation-aware path for stronger final quality with fallback semantics.

End-to-End Workflow

  1. Create project and start job.
  2. Monitor stage states and artifact generation.
  3. Edit score and checkpoint revisions.
  4. Review confidence and observability metrics.
  5. Export production-ready artifacts.

Observability & Quality Signals

  • Retain deterministic status payloads for reproducible triage.
  • Use performance tests to protect interaction latency budgets.
  • Enforce coverage gate with branch-awareness before release.
pytest --cov=. --cov-branch --cov-report=term-missing --cov-fail-under=95

Security & Privacy Controls

  • Authenticate by owner-scoped access token workflows.
  • Use signed artifact URL patterns.
  • Apply retention policies for retain/archive/delete outcomes.

Windows Local Runbook

Follow docs/runbooks/DT-016_Local_Dev_Windows_Runbook.md end-to-end. Validation commands:

  • Import collision: ensure --import-mode=importlib is active.
  • Coverage below gate: add branch-complete tests before merging.
  • Windows shell variance: execute from repository root using py -m ....
